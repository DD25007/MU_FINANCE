What this paper is about

The Core Problem
Credit scoring transformers are trained on millions of customer records. When a customer invokes GDPR Article 17 (Right to be Forgotten), the bank must remove their data's influence from the model. Full retraining takes hours or days. This paper proposes a fast, surgical alternative using LoRA adapters — reducing unlearning time from hours to minutes while provably satisfying forgetting criteria.
What LoRA is
Low-Rank Adaptation freezes pretrained weights W and learns two small matrices A and B such that the update is W + BA (rank r ≪ d). Only A and B are trainable. Originally for LLM fine-tuning — you are repurposing it for unlearning.
What Unlearning means here
Given a trained model M on dataset D, produce model M* that behaves identically to a model retrained from scratch on D minus the forget set D_f — but without actually retraining.
Why this combination is novel
LoRA has been used for fine-tuning and domain adaptation. Unlearning has been studied with gradient ascent, SISA, and influence functions. Nobody has combined LoRA's parameter-efficient update mechanism with the forgetting objective in a tabular financial model. That intersection is your contribution.
LoRA (Hu et al. 2021) Machine Unlearning (Bourtoule 2021) Credit Scoring Transformers GDPR Compliance
Your one-line contribution: "We propose LoRA-Credit-Unlearn — a parameter-efficient unlearning framework for transformer-based credit scorers that achieves certified forgetting in O(minutes) vs O(hours) for full retraining, with less than 1% AUC degradation."

The unlearning method — step by step

Three-phase framework
Phase 1: Train base model M on D → Phase 2: LoRA Forget Adapter on D_f → Phase 3: LoRA Retain Adapter on D_r → Merge → M*
Phase 1 — Train Base Credit Model
Standard
Train FT-Transformer on full dataset D = D_f ∪ D_r. Use binary cross-entropy loss for default prediction. This is your starting model M. Save all weights. This phase is done once — unlearning never touches these weights again.
Loss = BCE(M(x), y) for all x in D
Phase 2 — Forget Adapter (Gradient Ascent)
Core novelty
Freeze M. Attach LoRA adapter Θ_f to Q and V layers. Train ONLY Θ_f using gradient ascent on D_f — we want the model to perform poorly on forget-set samples, effectively erasing their signal. Run for a small number of steps (20–50) to avoid catastrophic drift.
Loss_forget = −BCE(M+Θ_f(x), y) for x in D_f
Key risk: too many gradient ascent steps will corrupt the model beyond D_f. Use early stopping on retain-set AUC to know when to stop.
Phase 3 — Retain Adapter (Knowledge Distillation)
Core novelty
Attach a second LoRA adapter Θ_r. Train it by distilling from the original frozen model M on the retain set D_r. The student (M + Θ_f + Θ_r) should match M's predictions on D_r exactly. This repairs any collateral damage from Phase 2 and locks in the useful knowledge.
Loss_retain = KL(M+Θ_f+Θ_r(x) ∥ M(x)) for x in D_r
Adapter Merging → Final Model M*
Output
Merge the retain adapter Θ_r into the base model weights: W* = W + BA for each adapted layer. The forget adapter Θ_f is discarded — its job was to create the signal for Phase 3. M* is your final unlearned model, deployable with no architectural changes.
W* = W_frozen + B_r · A_r (merge retain adapter only)
Optional extension: add a third adapter for fairness regularization — constrain equalized odds on protected attributes during Phase 3. This turns the paper into a fairness + unlearning dual contribution.

Full experimental pipeline

1
Data preparation
Load German Credit + GiveMeSomeCredit. Preprocess: one-hot encode categoricals, normalise numericals. Split: 70% train (D), 15% validation, 15% test. Within D, define D_f (forget set) using all 3 strategies above.
2
Train base model M
Train FT-Transformer + TabTransformer on full D. Record AUC, training time, model size. Save checkpoints. This is your starting point for all unlearning methods.
3
Run all baselines
Apply Full Retrain, SISA, Gradient Ascent, Influence Functions, and Fine-tune-on-D_r to the same base model. Record all 9 metrics for each. These go into your main results table.
4
Run your LoRA method
Apply Phase 2 (forget adapter) + Phase 3 (retain adapter). Sweep LoRA rank r ∈ {4, 8, 16}. Sweep forget-set size |D_f| ∈ {5%, 10%, 20%}. Record all 9 metrics for each configuration.
5
Ablation study
Remove Phase 3 (retain adapter) — does forgetting quality degrade? Remove Phase 2 (forget adapter) — does the model still forget? Vary LoRA rank. This section proves each component is necessary.
6
MIA attack experiment
Train a membership inference attack classifier on shadow models. Apply it to M* and to M_retrain. Show MIA score converges to 50% (random) for your method — this is your strongest proof of forgetting.
7
Scalability experiment
Repeat on GiveMeSomeCredit (150K) and LendingClub (temporal). Show wall-clock speedup scales with dataset size — larger datasets → greater advantage over full retrain.

What makes this publishable

The 3 novelty claims
Each claim must appear in your abstract and introduction:
N1
First LoRA unlearning for tabular credit models
LoRA has only been used for fine-tuning. Repurposing it for the unlearning objective (gradient ascent + distillation via adapters) is architecturally novel.
N2
Dual-adapter decomposition (forget + retain)
Separating the forget signal and the utility repair into two independent adapters is a design contribution. Prior gradient ascent methods do both in one uncontrolled pass.
N3
Finance-specific evaluation with regulatory framing
Applying MIA + Fairness ΔEO + GDPR compliance framing to credit unlearning is new. No existing paper benchmarks unlearning specifically against the EU AI Act's high-risk system criteria.
